# Design: "Potvrzeni o studiu" link in the Student flyout

## Problem

Students frequently need a "Potvrzeni o studiu" (confirmation of study enrollment) --
a common document required for things like transport discounts, health insurance,
or informal proof of enrollment. Today this requires navigating IS Mendelu
manually: Moje studium -> Tisk dokumentu -> find the right icon among several.
reIS should surface a direct link from its own Student flyout.

## IS Mendelu source page

`tisk_dokumentu.pl` (reached via Moje studium -> "Tisk dokumentu") offers two
distinct variants of this document, with materially different delivery
mechanics:

| Variant | URL param | Delivery |
|---|---|---|
| Instant, unsealed | `potvrzeni_tisk=1` | Returns a real PDF synchronously -- the browser can display it immediately in its native PDF viewer. No official electronic seal. |
| Sealed (e-pecet) | `potvrzeni_tisk_el=1` | Returns an HTML "request queued" confirmation, **not** a PDF. The sealed PDF is generated by IS asynchronously and appears up to an hour later in a separate app, Uloziste dokumentu (`/auth/uloziste/index.pl?rezim=2`). |

Both variants additionally support an English document-content variant via an
extra `jazyk=eng` param, independent of the `lang` param (which controls the IS
interface chrome language, not the document content).

## Decision

Use the **instant, unsealed** variant (`potvrzeni_tisk=1`). It's the only one
that can deliver a "click -> preview the PDF in a new tab, decide whether to
save it" experience, which is the desired UX. The sealed variant is explicitly
out of scope (see below) because its async, queued nature is incompatible with
that UX -- there's a real PDF to show or a "queued" HTML page, not both.

## Behavior

A new first entry in the `Student` flyout (`MainItems.tsx` -> the `is`
expandable item's `children` array), positioned before the existing Wi-Fi
(eduroam) entry. It's a plain `href`, identical in kind to the existing
entries in that list (Portal studenta, Zaznamnik ucitele, Testy) -- clicking it
opens a new tab straight to the IS URL; no popup, no async flow, no new
message-passing plumbing.

```
https://is.mendelu.cz/auth/student/tisk_dokumentu.pl?potvrzeni_tisk=1;studium={sid};lang={lang}
```

## Language handling

Two independent dimensions in IS's URL scheme, both derived automatically from
reIS's existing CZ/EN language state -- no new picker or setting:

- **`lang`** -- the IS interface language. Reused unchanged from what
  `mainItems(sid, oid, t, lang)` already receives and passes to every other
  entry in this list.
- **`jazyk=eng`** -- the language of the confirmation document's *content*.
  Appended only when reIS's current language is English (so the printed
  confirmation is in English); omitted for Czech (IS defaults to Czech
  content when the param is absent).

## i18n

New key `sidebar.confirmation`:
- cs: `"Potvrzeni o studiu"`
- en: `"Confirmation of Study"`

Follows the exact existing pattern of `sidebar.portal` / `sidebar.notebooks` /
`sidebar.tests` (flat key under the `sidebar` namespace, both locale files).

## Icon

`FileCheck2` (lucide-react) -- distinct from the existing
`LayoutDashboard`/`ClipboardList`/`PenTool` icons already used in this list,
and semantically fits "confirmation of standing."

## Data flow

No new data flow. This is a static, computed `href` string built from values
`MainItems.tsx` already receives as arguments (`sid`, `lang`) -- no fetch, no
store read, no IDB, no message passing. It renders through the same
`NavItem`/mobile-sheet code paths that already render the sibling entries in
this list.

## Error handling

None needed -- a plain anchor tag has no failure mode reIS is responsible for.
If the student's IS session has expired, IS's own login page will appear in
the new tab, exactly as happens today for the other flyout links (Portal
studenta, etc.).

## Testing

- Assert the new entry is first in the `is` item's `children` array.
- Assert the constructed `href` matches the expected pattern for both `lang`
  values, and that `jazyk=eng` is present only when `lang === 'en'`.
- Assert the i18n label renders from `sidebar.confirmation` in both locales.

## Explicitly out of scope

The **sealed** (e-pecet) variant. It requires a genuinely different,
asynchronous flow: trigger generation -> wait up to an hour -> retrieve from a
separate app (Uloziste dokumentu). This is incompatible with "preview then
download" and is not a small variation of this feature -- it would need its
own design (e.g., a status/notification affordance, a link to Uloziste
dokumentu) if ever pursued. Not building it now.
